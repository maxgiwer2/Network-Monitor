Build a real-time Network Monitoring System (NMS) as a single-page web application with the following requirements:

## Core Features

### 1. Network Topology Map (Main Canvas)
- Interactive node-link diagram showing all network devices as labeled nodes on a dark background (#001a0d or similar dark green/black)
- Each node is a small colored rectangle with a device name label
- Node colors indicate status:
  - Bright green = Normal / Device Responding to Poll
  - Red = Major alert / No Response to Device Poll
  - Yellow/Orange = Minor/Warning
  - Gray = Unknown/Unreachable
- Nodes are grouped by building/floor/zone (e.g., F1=Floor1, F2=Floor2, DC=DataCenter, AIR=AirCon etc.)
- Draw connection lines between linked devices to show network topology
- Pan and zoom support on the canvas (mouse wheel + drag)
- Click a node to show device details popup

### 2. Left Sidebar - Device Tree
- Collapsible tree panel listing all devices grouped by location/type
- Each device row shows a colored status dot
- Click to jump to / highlight that node on the map
- Search/filter box at top

### 3. Bottom Event Log Panel
- Scrollable table showing real-time events with columns:
  | Severity | Date | Time | Device | Message |
- Color-coded rows: green=Normal, red=Major, yellow=Minor
- Tab bar: Current | History | Custom 1–6
- Auto-scroll to latest event

### 4. Toolbar (Top)
- Buttons: Map | MIB | Trend | Event | [more]
- Zoom in/out controls
- Refresh interval selector
- Connection status indicator ("localhost | Administrator | Supervisor")

### 5. Real-time Polling Simulation
- Every 5–15 seconds, randomly change 1–3 nodes between Normal/Major/Minor
- When a node goes down: add a red "Major - No Response to Device Poll" event to log
- When it recovers: add a green "Normal - Device Responding to Poll" event

## Device Data (seed with these groups)
- Buildings: A-building, B-building, C-building, D-building, E-building, F-building, G-building
- Data Center: DC-DMZ, DC-emr_1, DC-emr_2, DC-emr_3, DC-net1, DC-net2, firewall, DC-S, DC-S-AP1, DC-S-AP2
- Floors F1–F16: switches, APs, ward nodes (e.g. F1_drug_S1, F2_jitavat_S1..S8, F7-R-1, F7-R-2, F9-ward)
- Special: OPD, EMR, X-Ray, Pharmacy, ICU, Lab, OR nodes
- Network gear: core1, core2, core3, Nac, AIR1, AIR2, printer

## Tech Stack
- Vanilla HTML + CSS + JavaScript (no frameworks required)
- Use HTML5 Canvas or SVG for the topology map
- Use CSS Grid/Flex for layout
- Optional: use D3.js for force-directed layout

## Layout
- Left panel: 180px fixed width, device tree
- Center: fills remaining width, topology canvas
- Bottom panel: 180px height, event log
- Top bar: 40px, toolbar

## Visual Style
- Dark theme: background #001505, panel bg #002a0c, border #00ff00 at low opacity
- Green phosphor aesthetic (inspired by classic network management tools like HP OpenView / Nagios / Observium)
- Node boxes: 14px font, white text, colored border matching status
- Connection lines: thin (#00aa44) between linked nodes
